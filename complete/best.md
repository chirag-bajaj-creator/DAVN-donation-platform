The real optimization hierarchy:

Quality of your prompt — A precise, well-scoped prompt gets the right answer first try. That's worth more than any token saving.
Right model for the job — Using Opus for a CSS fix is like using a sledgehammer on a nail. Not a token problem, a model-matching problem.
Context management — Projects > re-pasting. Not because it saves tokens but because Claude retains your codebase understanding across sessions.
Conversation structure — One focused conversation per task beats one giant thread where Claude loses track of the goal.
Token efficiency — Comes last. Only worth optimizing if you're hitting limits or using the API with billing.
