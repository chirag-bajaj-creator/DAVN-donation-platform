6. System Overview
Structure of Needy Section:
Needy Section
│
├── Organisation
│   ├── Registration
│   └── Check
│
└── Individual
    ├── Registration
    └── Check
________________________________________
7. Organisation Section
7.1 Registration (Organisation)
Description:
Organizations in need (e.g., NGOs, hospitals, schools) can register their requirements.
Input Details:
•	Organisation Name
•	Phone Number
•	Address
•	Type of Need (food, funds, medical, shelter, etc.)
•	Description of requirement
Workflow:
1.	Organisation fills registration form
2.	Submits request
3.	Data is stored in the system
4.	Request is forwarded to Admin Panel
________________________________________
7.2 Verification Process
 

 

 
Description:
Ensures that only genuine organizations are approved.
Steps:
1.	Admin receives registration request
2.	Admin assigns a volunteer for verification
3.	Volunteer checks the organization anonymously
4.	Volunteer submits verification report
5.	Admin approves or rejects the request
Key Feature:
•	Prevents fraud and ensures authenticity
________________________________________
7.3 Check (Organisation)
Description:
Displays verified organizations to donors.
Categories of Organisations:
•	🏥 Hospitals
•	🏫 Schools
•	🧒 Orphanages
•	🌪 Disaster-affected areas
Information Displayed:
•	Organisation Name
•	Location
•	Type of Need
•	Description
Key Feature:
•	Only verified organizations are shown
________________________________________
8. Individual Section
8.1 Registration (Individual)
Description:
Individuals in need can register themselves for assistance.
Input Details:
•	Name
•	Phone Number
•	Address
•	Type of Need
•	Description of situation
Workflow:
1.	Individual fills the form
2.	Submits request
3.	Data goes to Admin Panel for verification
________________________________________
8.2 Verification Process (Individual)
Steps:
1.	Admin reviews request
2.	Volunteer is assigned
3.	Volunteer verifies the individual situation
4.	Report submitted to admin
5.	Admin approves/rejects
Key Feature:
•	Ensures help reaches genuine individuals
________________________________________
8.3 Check (Individual)
Description:
Displays verified individuals who need help.
Information Displayed:
•	Name
•	Location
•	Type of Need
•	Situation Description
Key Feature:
•	Transparent listing of verified needy people
________________________________________
9. Data Flow (Needy Module)
Registration → Admin Panel → Volunteer Verification → Approval/Rejection → Display in Check Section
________________________________________
10. Key Features of Module
•	Dual structure: Organisation & Individual
•	Registration + Verification system
•	Admin-controlled approval process
•	Volunteer-based ground verification
•	Transparent “Check” listing
•	Categorized needy data
